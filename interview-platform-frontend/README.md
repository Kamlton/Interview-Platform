# Платформа для технических собеседований — Frontend (React)

SPA-фронтенд к API платформы технических собеседований. Стек: **React 18 +
TypeScript + Vite**, маршрутизация — react-router, HTTP — axios. Дизайн —
единая визуальная система (navy/slate), общая с проектной документацией.

> Это **каркас** под готовый backend: авторизация с ролями, реестры с поиском
> и пагинацией, карточки, заполнение оценок, решение и скачивание печатных форм
> реализованы. Команда расширяет по тем же шаблонам.

## Предварительные требования

- Node.js 18+ (рекомендуется 20+)
- Запущенный backend (по умолчанию `http://localhost:5080`)

## Быстрый старт

```bash
npm install
npm run dev
```

Откройте `http://localhost:5173`. В dev-режиме запросы `/api/*` проксируются
на backend (`vite.config.ts`), поэтому CORS настраивать не нужно.

Демо-доступ (сид backend): **admin@local / Admin#12345**.

### Сборка

```bash
npm run build      # tsc --noEmit + vite build -> dist/
npm run preview    # предпросмотр собранной версии
```

## Конфигурация

`.env` (см. `.env.example`):

- `VITE_API_BASE` — базовый URL API. В dev оставьте пустым (работает прокси Vite).
  Для прод-сборки укажите адрес API, например `https://api.example.com`.

## Что реализовано

| Экран | Файл | Доступ |
|---|---|---|
| Вход | `pages/LoginPage.tsx` | Все |
| Реестр собеседований | `pages/InterviewsPage.tsx` | Авторизован |
| Карточка собеседования (оценки, протокол, решение, печать) | `pages/InterviewPage.tsx` | Авторизован; решение — Решала/Админ |
| Реестр кандидатов (поиск, пагинация) | `pages/CandidatesPage.tsx` | HR, Админ |
| Карточка кандидата (навыки, история) | `pages/CandidateFormPage.tsx` | HR, Админ |
| Вакансии | `pages/VacanciesPage.tsx` | HR, Админ |
| Компетенции | `pages/CompetenciesPage.tsx` | HR, Админ |
| Пользователи | `pages/UsersPage.tsx` | Админ |

Ключевые механизмы:

- **JWT**: токен хранится в localStorage, подставляется в каждый запрос
  (`api/client.ts`); на 401 — выход на страницу входа.
- **Роли**: `AuthContext` + `ProtectedRoute` ограничивают маршруты; навигация
  и кнопки скрываются по роли.
- **Печатные формы**: скачиваются авторизованным blob-запросом (с JWT), а не
  открытием ссылки, поэтому корректно проходят авторизацию.

## Структура

```
src/
├── main.tsx, App.tsx          // вход и маршрутизация (защита по ролям)
├── index.css                  // дизайн-система (палитра, таблицы, формы, состояния)
├── api/
│   ├── client.ts              // axios + JWT-перехватчик + декодер токена
│   └── index.ts               // методы всех эндпоинтов
├── types/index.ts             // типы — зеркало DTO бэкенда
├── auth/                      // AuthContext, ProtectedRoute
├── components/                // Layout, UI-компоненты (бейджи, пагинация, состояния)
└── pages/                     // экраны
```

## Упрощения каркаса (точки роста)

- Оценки в карточке собеседования выставляются по **всему каталогу
  компетенций**; привязку к конкретной матрице (через `MatrixId`) можно
  добавить, когда появится эндпоинт списка матриц.
- Интервьюером назначается **текущий пользователь** (id берётся из JWT).
  Для выбора другого интервьюера нужен доступ к списку пользователей.
- Нет страницы «Настройка прав» (модель прав на бэкенде уже есть).
- Валидацию форм можно усилить (react-hook-form + zod); сейчас — нативная.
